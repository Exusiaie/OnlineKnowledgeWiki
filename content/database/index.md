# 数据库概述

## 什么是数据库

数据库是有组织地存储和管理数据的电子系统。它不仅仅是数据的仓库，更是数据安全、完整性和高效访问的保障。无论是电商平台的订单管理、社交媒体的用户信息，还是银行系统的交易记录，都离不开数据库的支持。

## 数据库分类

### 关系型数据库（RDBMS）

关系型数据库使用表（Table）来组织数据，通过 SQL（Structured Query Language）进行查询和管理。数据按行和列存储，表之间通过外键建立关联。

**特点**：
- 结构化数据模型，模式（Schema）预先定义
- 支持 ACID 事务（原子性、一致性、隔离性、持久性）
- 强大的 JOIN 查询和复杂查询能力
- 严格的完整性约束

**代表**：MySQL、PostgreSQL、SQLite、Oracle、SQL Server

### NoSQL 数据库

NoSQL（Not Only SQL）是非关系型数据库的统称，设计用于处理非结构化、半结构化数据和大规模分布式场景。

**类型与代表**：
- **文档型**：MongoDB、CouchDB（JSON/BSON 文档存储）
- **键值存储**：Redis、DynamoDB（简单快速的 K-V 操作）
- **列族存储**：Cassandra、HBase（宽列存储）
- **图数据库**：Neo4j、Amazon Neptune（实体与关系分析）

## SQL vs NoSQL

| 维度 | SQL | NoSQL |
|------|-----|-------|
| 数据模型 | 预定义模式、表结构 | 灵活模式、多种模型 |
| 扩展方式 | 垂直扩展（升级硬件） | 水平扩展（增加节点） |
| 事务 | 强大 ACID 支持 | 部分支持（最终一致性） |
| JOIN | 原生复杂 JOIN | 通常不支持或有限支持 |
| 适用场景 | 结构化数据、金融、ERP | 大数据、实时分析、社交 |

## ACID 理论

ACID 是关系型数据库事务的四大特性：

- **A（Atomicity 原子性）**：事务是一个不可分割的单元，要么全部执行，要么全部不执行
- **C（Consistency 一致性）**：事务执行前后，数据必须保持一致状态
- **I（Isolation 隔离性）**：并发事务之间相互隔离，互不干扰
- **D（Durability 持久性）**：一旦事务提交，数据永久保存，即使系统崩溃

## CAP 理论

在分布式系统中，CAP 定理指出：一致性（Consistency）、可用性（Availability）和分区容错性（Partition Tolerance）三者最多只能同时满足两个。

- **CP 系统**：优先一致性和分区容错（如传统 RDBMS 集群）
- **AP 系统**：优先可用性和分区容错（如 Cassandra）
- **CA 系统**：优先一致性和可用性（单节点 RDBMS）

## 索引与性能

索引是提升查询性能的关键手段，类似书籍的目录：

- **B-Tree 索引**：最常用的索引结构，适合范围查询
- **哈希索引**：适合等值查询
- **全文索引**：文本搜索
- **空间索引**：地理位置搜索

## 推荐学习路径

对于数据库学习的初学者：

1. **MySQL**：最流行的开源 RDBMS，SQL 入门首选
2. **PostgreSQL**：功能最丰富，学习高级 SQL 特性
3. **MongoDB**：文档型 NoSQL，理解非关系型思维
4. **Redis**：高性能缓存，掌握内存数据操作
5. **SQLite**：嵌入式数据库，适合移动端和小型应用

## 总结

从单体应用到分布式系统，从传统 RDBMS 到 NoSQL 混合架构，数据库技术一直在演进。理解 SQL 和 NoSQL 各自的适用场景，掌握一到两种数据库的深入使用，是后端开发者的基本功。扎实的 SQL 功底和数据库设计能力，是写出高性能应用的关键。
